# BabyBite Sanctuary: Project Roadmap & Next Steps

Now that we have the core functional screens (Calendar, Library, and Recipe Planner) designed for desktop, we can shift focus to refining the user journey and preparing for development.

## 1. Responsive Design (Mobile First)
Parents are often multi-tasking. We need to ensure the **Calendar View** and **Food Library** are highly usable on mobile devices.
*   **Task:** Create mobile versions of the Dashboard and Food Library.
*   **Focus:** Thumb-friendly navigation, collapsible inspectors, and clear status indicators.

## 2. Daily Entry & Interaction
The current calendar is a "Planner." We need a "Logging" mode for when the meal actually happens.
*   **Task:** Design a **Daily Entry Screen**.
*   **Features:** Toggle "Offered" vs "Eaten," note potential allergic reactions (redness, hives), and upload a photo of the "clean plate" or "messy face."

## 3. Onboarding & Personalization
How does the app know when to start?
*   **Task:** Design an **Initial Setup Flow**.
*   **Features:** Set the child's name/age, select the start date (March 28, 2026), and identify any known family allergies to prioritize or avoid.

## 4. Export & Utility
The document mentions CSV and Print/PDF export.
*   **Task:** Design a **Print-Optimized Calendar View**.
*   **Focus:** A high-contrast, ink-friendly version of the month view that looks great pinned to a refrigerator.

## 5. Logic & Validation Edge Cases
Refining the "Rules Engine" UI.
*   **Task:** Design **Conflict Resolution Modals**.
*   **Scenario:** What happens if a user manually drags "Peanut" to a day that violates the 3-day allergen gap? We need a clear way to explain the rule and suggest an alternative date.
